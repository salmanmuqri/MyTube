## 3. Project Summary

### 3.1 Problem

Build a self-hostable video platform that provides:
- video upload and adaptive playback
- user authentication and subscriptions
- personalized recommendations
- analytics and trend discovery
- admin moderation controls

### 3.2 Solution

MyTube combines:
- Frontend: React + Vite
- Backend: Django + DRF
- ML: content-based + behavior-based recommendation logic
- Processing: asynchronous/scheduled jobs for ranking and model refresh
- Deployment: split hosting (frontend on Vercel, backend on Railway)

### 3.3 Core Deliverables

- HLS-ready video playback workflow
- personalized recommendation endpoints
- trending ranking pipeline
- admin panel APIs and controls
- production deployment hardening via environment-driven config

## 6. Machine Learning Models And Algorithms (Primary Focus)

This section is intentionally detailed to support ML-heavy reporting.

### 6.1 Recommendation Strategy

Hybrid recommendation architecture:
- content-based retrieval for semantic relevance
- implicit behavioral scoring from watch interactions
- weighted rank fusion for final ordering

### 6.2 Content-Based Retrieval

Feature text sources:
- title
- description
- tags
- category

Pipeline:
1. normalize text
2. tokenize and vectorize with TF-IDF
3. compute cosine similarity between items
4. retrieve top-N similar items

Similarity definition:

similarity(i, j) = (v_i dot v_j) / (||v_i|| ||v_j||)

where v_i and v_j are TF-IDF vectors.

### 6.3 Behavioral / Collaborative Signal Layer

Implicit feedback features:
- watch duration
- completion percentage
- repeat views
- recency

A practical interaction confidence model:

pref(u, i) = a * watch_time_norm + b * completion_norm + c * repeat_norm + d * recency_weight

This avoids dependence on explicit star ratings.

### 6.4 Hybrid Score Fusion

Final ranking can be expressed as:

final_score(u, i) = w1 * content_score + w2 * behavior_score + w3 * popularity_score + w4 * freshness_score

Interpretation:
- w1 stronger for sparse-history users
- w2 stronger as user history grows
- w3/w4 stabilize quality and recency

### 6.5 Trending Algorithm

Trending is treated as a momentum-quality ranker:

trending(i) = alpha * views_norm + beta * likes_norm + gamma * retention_norm - delta * age_penalty

Benefits:
- rewards engaging recent content
- avoids permanent domination by old viral items

### 6.6 Training / Scheduling

- periodic jobs recompute trend/recommendation signals
- retraining cadence can be daily for model freshness
- fast trend updates can run hourly

### 6.7 Cold Start Strategy

User cold start:
- start with trending + category-diverse picks
- adapt rapidly from first session behavior

Item cold start:
- content-based vectors available immediately from metadata
- no wait for long interaction history

### 6.8 Evaluation Framework

Offline metrics:
- Precision@K
- Recall@K
- NDCG@K
- coverage/diversity

Online metrics:
- recommendation CTR
- watch-time uplift
- completion-rate uplift
- session-length uplift

Quality checks:
- popularity bias drift
- duplicate-heavy list detection
- topic diversity constraints

### 6.9 Explainability

Recommended explanation text for trust and UX:
- Because you watched <video>
- Similar topic: <tag/category>
- Trending in your interest profile

## 8. Deployment (Vercel + Railway)

### 8.1 Deployment Topology

- frontend: Vercel
- backend: Railway
- browser calls backend APIs directly through configured base URL

### 8.2 Critical Environment Variables

Backend:
- DEBUG=0
- ALLOWED_HOSTS includes Railway domains
- CORS_ALLOWED_ORIGINS includes production frontend origin(s)
- CORS_ALLOWED_ORIGIN_REGEXES can allow preview subdomains
- CSRF_TRUSTED_ORIGINS set for trusted browser origins
- ENABLE_MEDIA_SERVING=true when backend serves media directly

Frontend:
- VITE_API_BASE_URL points to backend and should resolve to /api path
- VITE_MEDIA_BASE_URL points to backend /media path (recommended)

### 8.3 Common Production Failure Patterns

No login/signup/videos/admin:
- wrong API base URL path or CORS block

Images/videos broken while API works:
- media URLs resolved to frontend host instead of backend host

400 host errors:
- backend host not included in ALLOWED_HOSTS
